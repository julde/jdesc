BufferPool buffer management GUI



BufferPool provides an easy way to load and organise soundfile buffers. Any number of files can be loaded and saved to disk as a pool (the paths not the soundfiles) . Furthermore collections of pools can be saved as presets. The settings are stored in preferences/ pools.bpl and presets.bpl respectively. Buffers are allocated when a pool is loaded and freed when removed. Currently when a soundfile appears in more than one place (in different pools), it is allocated a different buffer.


Use at own risk. This class has not been extensively tested, so I wouldn't recommend over-depending on it for organising complicated projects yet. Organise your soundfiles in folders as well, so that you can easily reload them and always backup the .bpl files.


To run:


BufferPool.new;


Class Methods


*instance

Returns the instance of BufferPool. Only one instance can exist at a time.


*bounds(a Rect)

Set the position of the window.


*backgroundColor_(a Color)

Set the background Color.


*buttonColor_(a Color)

Set the buttons Color.


*buttonTextColor_(a Color)

Set the button text Color.


*listsColor_(a Color)

Set the list background Color.


*textBoxColor_(a Color)

Set the Color for the text box.


*editor_(a String)

Set a program to open the soundfile on double click. Assumes that the program is in the Applications folder. 

Default is "QuickTime Player"


BufferPool.editor_("Wave Editor"); //set Wave Editor


Instance Methods


get(a Symbol)

Get the List containing the buffers of a pool.


bounds

Post the current position of the window.


BufferPool.instance.bounds;


currentPools

Posts the current pools formated to copy and paste in your code.


BufferPool.instance.currentPools;


loadPreset(a Symbol)

Load a preset by name. Put it in your startup file to have your project ready to go.


BufferPool.instance.loadPresets('name');


Examples



Add file(s) at the buffers menu (1) will open a dialog to choose the soundfiles. The soundfiles can then be saved as a pool which will cause the actual buffers to be allocated. At this point you can access a pool by name:


 BufferPool.instance.get('iceCrack'); //will return the List of buffers

 BufferPool.instance.get('iceCrack')[0]; //will return the buffer at Ice_crack_01.wav


All pools you have created can be accessed and loaded using load at the pools menus (2). To create a new pool use clear to empty the buffer list (only in appearence. The buffers are still there) and add the new files. If you dont use clear you can add or remove files to an existing pool but the changes will have effect only when you press overwrite (2). 


Finally you can save a collection of pools as a preset by pressing new (3) and choosing a name. Again any changes you make to the current pool will not be saved untill you press overwrite (3).


Tip: double click on a soundfile name to open it with your favourite program (See *editor_() class mehod).
